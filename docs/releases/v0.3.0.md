# Release Notes - Version 0.3.0

**Release Date**: November 26, 2025

## Overview

Version 0.3.0 is a maintenance release focused on improving error handling and updating project maintainer information. This release resolves Issue #1 where generic error messages were being displayed instead of actual API errors.

- **Solution**: Enhanced error handling to show detailed, contextual error messages

### ‚ú® Enhancements

#### Enhanced APIError Exception
- Added `request_method` attribute to track HTTP method (GET, POST, etc.)
- Added `request_url` attribute to show the full URL that was requested
- Improved `__str__()` method for better error representation
- Error messages now include:
  - Actual API error message
  - Additional error details from the response
  - Request method and endpoint
  - HTTP status code

#### Better Error Categorization
- **Connection Errors**: Now specifically identified with "Connection failed to [URL]"
- **Timeout Errors**: Now show "Request timeout after [X]s"
- **API Errors**: Show full context with status codes and request details

## Technical Details

### Error Message Format

New error messages follow this format:

```
[API Error Message] | Details: [Additional Info] | Request: [METHOD] [ENDPOINT]
```

### Exception Attributes

The `APIError` exception now provides:

```python
try:
    client.accounts.get(account_id=123)
except APIError as e:
    print(e.status_code)      # HTTP status code (e.g., 404)
    print(e.request_method)   # HTTP method (e.g., "GET")
    print(e.request_url)      # Full URL requested
```

### Error Handling Example

```python
from sipnav import SipNavClient
from sipnav.exceptions import APIError, AuthenticationError, SipNavException

try:
    client = SipNavClient(api_key="token")
    account = client.accounts.get(account_id=99999)
except AuthenticationError as e:
    print(f"Auth failed: {e}")
except APIError as e:
    print(f"API Error: {e}")
    print(f"Status: {e.status_code}")
    print(f"URL: {e.request_url}")
except SipNavException as e:
    print(f"Error: {e}")
```
